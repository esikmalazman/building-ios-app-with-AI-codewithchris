# Build an App with AI using Cursor

This repository contains a project to help myself learn how to building iOS app and communicate visual design changes using Cursor AI.
The goal is to help me to practice to adopt my workflow and effectively describing design improvements to AI.

## Repository structure
```
TEST-CURSOR/
├── test-cursor/                  # Main working directory for the app
│   │   ContentView.swift         # Default file generated by Xcode when you create a new SwiftUI project
│   │   test_cursorApp.swift      # Entry point of the app
│   └── Assets.xcassets/          # Folder for app assets i.e image, icon
│   └── instructions/
│   │   ├── instructions.md       # Instructions file for AI to reference
│   └── .cursorrules              # Context and guidlines for AI to follow
├── test-cursor.xcodeproj/        # Xcode project configuration file 
└── buildServer.json/             # File generated by Sweetpad - allow to select Xcode Scheme and Destination to run the project
└── README.md                     # Project instructions and requirements
```



## Tips for communicating development changes

1. Do something - Use this to implement features.
2. Give me options - Use this to have some control over how things are done.
3. Tell me info - Use this to ask for advice and to learn.
4. Screenshots - Use visuals to give Al context to what you're talking about.
5. Troubleshooting - Use these prompts to fix your project.


## Tips for communicating design changes

When asking Cursor to help improve the visual design:

1. Be specific about what aspects need improvement
2. Use clear design terminology
3. Reference specific elements and their relationships
4. Provide context about the desired look and feel
5. Break down complex changes into smaller steps

Example request:
```
Could you help improve the visual hierarchy of this card component by:
1. Adding more breathing room between elements
2. Making the title more prominent
3. Softening the contrast of secondary text
```
